# The token MUST be in the password position (x-access-token:<token>@). With the
# token as username only (<token>@), modern git treats the password as missing
# and tries to prompt for it — which fails non-interactively in Docker builds
# ("could not read Password ... No such device or address"). Keep the
# x-access-token: prefix. Applies to both `pip install -r` (pip expands ${...})
# and the Docker subst-sdk-requirements.py path.
